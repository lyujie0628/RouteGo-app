# 🚀 走起 · 地图驱动的旅行路线设计工具

一个基于地图交互的旅行路线设计 Web App  
核心目标：让用户像“搭积木一样”自由规划旅行路线，而不是被动接受导航结果

## 📌 项目简介

在现有地图产品（如高德地图）中，用户通常只能进行“点对点导航”或使用固定推荐路线，缺乏对整个行程的灵活控制能力。

本项目从零开始设计并实现了一个“旅行路线设计工具”，允许用户在地图上自由选点、构建行程、调整顺序并进行整体优化。

相比传统地图产品，本项目更关注：

- 路线的“可编辑性”
- 行程的“整体规划能力”
- 用户的“主动控制体验”

定位为：

👉 路线设计工具，而不是导航工具

---

## ✨ 核心功能

### 1. 地图选点 + 行程构建

- 点击地图添加景点
- 自动解析地点名称（逆地理编码）
- 实时生成行程列表

---

### 2. 行程编辑系统（核心模块）

- 支持拖拽排序（Drag & Drop）
- 支持修改景点名称
- 起点 / 终点锁定机制
- 行程结构实时更新

该模块本质上实现了一个“可操作的路线数据结构”，是整个系统的核心

---

### 3. 多交通方式路线规划

支持三种交通方式：

- 步行
- 骑行
- 驾车

实现方式：

- 按段（segment）进行路径计算
- 调用地图 API 获取路径
- 拼接为完整路线

---

### 4. 路线缓存机制（性能优化）

为减少 API 调用成本并提升响应速度：

- 使用 localStorage 做本地缓存
- 以「起点 + 终点 + 交通方式」作为缓存 key
- 优先复用历史路径数据

---

### 5. 时间轴系统（行程规划能力）

支持时间维度的规划：

- 设置行程开始时间
- 每个景点可设置：
  - 到达时间
  - 停留时间

系统自动计算：

- 各节点时间
- 行程流转逻辑

并通过时间轴进行展示

---

### 6. 一键优化路线

- 自动调整景点顺序
- 保留已锁定的起点 / 终点
- 输出更合理路径结构

---

### 7. 状态持久化

- 自动保存行程（localStorage）
- 页面刷新后恢复：
  - 路线
  - 城市位置
  - UI状态

提升整体使用连续性

---

## 🧩 技术架构

前端：

- HTML / CSS / JavaScript（原生开发）
- 模块拆分：
  - state.js（数据管理）
  - map.js（地图与路径逻辑）
  - ui.js（界面交互）
  - data.js（数据结构）

地图服务：

- 高德地图 JS API
  - 路径规划（Driving / Walking / Riding）
  - 地理编码（Geocoder）

## 🧠 设计思考

### 为什么不做“攻略 / 社区”？

当前市场已经存在大量成熟产品（如小红书、马蜂窝等）提供：

- 景点评价
- 攻略内容
- 用户分享

本项目选择不进入该方向，而是聚焦一个更纯粹的能力：

👉 帮助用户“设计路线”

避免功能膨胀，专注解决一个核心问题

---

### 产品核心理念

不是告诉用户“去哪玩”，而是：

👉 帮助用户“怎么把这些地方连起来”

---

### 用户价值

相比传统地图工具，本产品提供：

- 更高自由度（任意加点）
- 更强可控性（手动调整 + 锁定）
- 更完整视角（从“点”到“路线”）

---

## 📈 项目亮点

- 从 0 到 1 独立设计并实现完整产品
- 将“地图能力”转化为“路线设计能力”
- 构建可编辑路线系统（非单次导航）
- 引入缓存机制优化 API 成本
- 支持时间维度的行程规划
- 实现接近真实产品体验（非 Demo）
